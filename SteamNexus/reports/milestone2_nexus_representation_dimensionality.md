# Milestone 2: Representation and Dimensionality Report - Steam Nexus

## 1. Objective

This milestone focuses on moving from raw integrated tables to mathematical and technical representations suitable for advanced analysis. The goal is to build reproducible feature matrices, apply dimensionality reduction techniques, and compare the performance of numeric and text-based representations. This work is designed to prepare the project for the upcoming clustering and NLP modeling stages.

## 2. Data and Scope

**Dataset:** `data/processed/steam_games_reviews_integrated_sample_v1.csv`

**Working sample:** Integrated games and reviews data following the ERD relationship `GAMES.AppID = REVIEWS.app_id`.

**Dataset size:**
- Unique games in the integrated dataset: 14,154
- Games with reviews: 1,056
- Review counts are skewed, with median 57 and max 88,781.

**Filtering strategy:** The original integrated dataset contained ~900k rows. For this milestone, it was reduced to the 1,056 games with review data in order to focus on higher-quality entries and improve computational efficiency. The selected subset contains the most reviewed titles, which are more informative for representation learning and reduce complexity.

## 3. Feature Engineering

### 3.1 Numeric Features

Numeric features were derived from the integrated game and review data, including:
- `Price`
- `Positive`
- `Negative`
- `Review_count`
- `avg_review_score`
- `sum_review_votes`
- `avg_review_votes`
- `Release_year`
- `Genre_count`
- `Tag_count`

These features were normalized with `StandardScaler` before applying dimensionality reduction.

### 3.2 Text Features

Text representations were built using:
- `Genres`
- `Tags`
- Aggregated `review_text` per game

The text content was combined into a single document per game and vectorized using `TfidfVectorizer` with English stop-word removal and unigrams/bigrams.

### 3.3 Dataset Filtering

The analysis pipeline keeps only games with review data and selects the most popular titles by review count. This reduces the sample from the original integrated dataset and keeps the analysis focused on high-value entries.

## 4. Dimensionality Reduction

### 4.1 PCA on Numeric Features

Principal Component Analysis (PCA) was applied to the scaled numeric feature matrix with 10 components. Key outputs:
- Explained variance per component
- Cumulative variance curve
- 2D PCA scatter plot using the first two principal components

### 4.2 SVD on Text Features

Truncated SVD was applied to the TF-IDF matrix with 10 components. Key outputs:
- Explained variance per component
- Cumulative variance curve

### 4.3 Optional t-SNE Visualization

For additional visual inspection, t-SNE projections were generated for:
- Numeric features (scaled data)
- Text features (SVD output)

These projections help visualize the structure of the selected game subset in 2D.

## 5. Evaluation and Comparison

A comparison table was produced to evaluate:
- Number of components
- Cumulative variance retained in the first 3 components
- Reconstruction Mean Squared Error (MSE)

This comparison provides a quantitative basis for choosing representation techniques in later stages.

## 6. Exports and Artifacts

### Files generated by the pipeline
- `data/processed/steam_representation_features_v1.csv`
  - Contains the final numeric feature matrix for the selected games.
  - Includes derived features and PCA projections (`PC1`, `PC2`).

- `data/processed/steam_selected_games_reviews_v1.csv`
  - Contains the raw integrated review rows for the selected games used in this analysis.
  - Ensures reuse of the same game subset for downstream modeling.

## 7. Key Results

- The filtered subset focuses on games with actual review history, providing higher-quality data for analysis.
- PCA on numeric features captured a substantial fraction of variance in its first components.
- SVD on text features captured semantic patterns in game genre/tag/review text, although text variance is more distributed.
- Reconstruction error confirms that the numeric PCA pipeline is highly faithful, while the text SVD pipeline is appropriate for sparse document data.

## 8. Reproducibility

The analysis is reproducible from the notebook:
